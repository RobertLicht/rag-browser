    RAG-Browser — About

    Overview

    A fully client-side, browser-based Retrieval-Augmented Generation (RAG) agent. Upload documents (.txt, .md, .csv, .xls, .xlsx, .docx, .pptx, .odt, .ods, .odp, .pdf), embed them locally, and query them conversationally — all without a server, API keys, or cloud infrastructure.

    No user data ever leaves your device. All inference runs locally in your browser.

    Model Storage & Cache

    Where is the model stored?

    The models are cached in your browser's storage:

    1. Open Developer Tools (F12 or Ctrl+Shift+I)
    2. Go to the Application tab
    3. Expand Storage → IndexedDB
    4. Look for databases containing transformers or huggingface
    5. Model files are stored in Cache Storage under transformers-cache

    How to delete the model cache?

    Method 1 — Via Developer Tools:

    1. Open Developer Tools (F12)
    2. Go to Application tab
    3. Click Storage → Clear storage
    4. Check Cache storage and IndexedDB
    5. Click Clear site data

    Method 2 — Via Browser Settings:

    1. Go to browser Settings → Privacy & Security
    2. Click Clear browsing data
    3. Select Cached images and files
    4. Choose time range and clear

    Method 3 — Via Console:

    1. Open Console in Developer Tools
    2. Run: caches.keys().then(keys => keys.forEach(k => caches.delete(k)))

    Enable WebGPU

    Important: WebGPU significantly improves performance. Without it, the model runs on CPU which can be 10–50× slower.

    Google Chrome & Microsoft Edge (Chromium)

    WebGPU is enabled by default starting with Chrome 113 and Edge 113. If it is not available:

    1. Navigate to chrome://flags (or edge://flags)
    2. Search for WebGPU
    3. Set Unsafe WebGPU to Enabled
    4. Click Relaunch

    Command Line Alternative: --enable-unsafe-webgpu

    Mozilla Firefox

    WebGPU is enabled by default from Firefox 141+ (Windows) and 145+ (macOS). To enable manually:

    1. Type about:config in the address bar and accept the warning
    2. Search for dom.webgpu.enabled
    3. Toggle to true
    4. Optionally set gfx.webgpu.ignore-blocklist to true if blocked

    Safari

    WebGPU is enabled by default starting with Safari 26 (macOS Tahoe 26, iOS 26). Earlier versions require Safari Technology Preview.

    Linux (Chrome/Chromium)

    Launch with the following flags:

    --enable-unsafe-webgpu --ozone-platform=wayland --use-angle=vulkan --enable-features=Vulkan,VulkanFromANGLE

    Ensure your graphics drivers are up-to-date. You may also try --ozone-platform=x11 instead of Wayland.
